# Contributing

## Commit messages

Use the [Conventional Commits](https://www.conventionalcommits.org/) format:

```text
<type>(<optional scope>): <description>
```

Keep the description concise, imperative, and lowercase. Use a scope when it
helps identify the affected area.

Examples:

```text
fix(scene): unify directional label positioning
feat: add matrix playback controls
docs: document commit requirements
chore(release): 1.0.7
```

Common types include `feat`, `fix`, `docs`, `refactor`, `test`, `chore`, and
`build`. Mark breaking changes with `!` before the colon or a `BREAKING CHANGE:`
footer.
